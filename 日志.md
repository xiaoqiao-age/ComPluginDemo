### 一、项目概述

本项目是一个 **Reg-Free COM (免注册 COM)** 插件系统，基于 Windows SxS (Side-by-Side) 清单机制，实现不依赖 `regsvr32`/`regasm` 注册就能加载 COM 组件。

#### 1.1 技术栈

| 组件 | 技术 | 说明 |
|------|------|------|
| ComPluginHost | C# .NET 8 + Avalonia 11.2.3 | 宿主程序，x86，MVVM 架构 |
| CSharpComPlugin | C# .NET 8 COM | Calculator 插件，`<EnableComHosting>` 生成 `.comhost.dll` |
| VB6ComPlugin | VB6 ActiveX DLL | StringProcessor 插件，GBK 编码 |

#### 1.2 核心 COM 接口

```csharp
// src/CSharpComPlugin/IComPlugin.cs
[ComVisible(true)]
[Guid("8C5E2B1A-3F4D-4A6E-9B7C-1D2E3F4A5B6C")]
[InterfaceType(ComInterfaceType.InterfaceIsDual)]  // 同时支持 vtable 和 IDispatch
public interface IComPlugin
{
    [DispId(1)] string Name { get; }
    [DispId(2)] string Version { get; }
    [DispId(3)] string Description { get; }
    [DispId(4)] void Initialize();
    [DispId(5)] string Execute(string input);
    [DispId(6)] void Shutdown();
}
```

- `InterfaceIsDual`: VB6 晚期绑定走 IDispatch (DispId)，早期绑定走 vtable
- Calculator CLSID: `{D7E8F9A0-1B2C-3D4E-5F6A-7B8C9D0E1F2A}`
- VB6 StringProcessor CLSID: 每次编译变化，由 `deploy_vb6_plugin.py` 从注册表读取

#### 1.3 插件配置

```json
// src/ComPluginHost/plugins.json
{
  "Plugins": [
    { "ProgId": "ComPluginDemo.Calculator", "ManifestPath": "CSharpComPlugin\\CSharpComPlugin.manifest" },
    { "ProgId": "VB6ComPlugin.StringProcessor", "ManifestPath": "VB6ComPlugin\\VB6ComPlugin.manifest" }
  ]
}
```

### 二、功能一：VB6 窗体 + C# COM 值传递

#### 2.1 数据流设计

```
C# Host (Avalonia)
  │
  ├─ 1. LoadPlugin("ComPluginDemo.Calculator")      → Activation Context → 创建 COM 对象
  ├─ 2. LoadPlugin("VB6ComPlugin.StringProcessor")   → Activation Context → 创建 COM 对象
  ├─ 3. PassCalculatorToVB6():
  │     dynamic vb6Obj = vb6Plugin.ComObject;
  │     vb6Obj.SetCalculatorObject(calculator.ComObject);  // 跨 COM 对象传递
  │     VB6 通过 late binding 读取 Calculator 的 Name/Version/Description 并缓存
  │
  └─ 4. 用户输入 "show" → vb6Obj.Execute("show")
        └─ VB6 创建 frmCalculator 窗体
           ┌─────────────────────────────────┐
           │ C# COM Component Info           │
           │ Connected: Calculator v1.0.0    │
           │─────────────────────────────────│
           │ Name:        [Calculator      ] │
           │ Version:     [1.0.0           ] │
           │ Description: [数学表达式计算器 ] │
           │─────────────────────────────────│
           │ Expression:  [1+2*3    ] [Calc] │
           │ Result:      [7        ]        │
           │                         [Close] │
           └─────────────────────────────────┘
```

#### 2.2 C# 宿主端：PassCalculatorToVB6()

```csharp
// src/ComPluginHost/ViewModels/MainWindowViewModel.cs
// 在 LoadAllPlugins() 末尾调用

private void PassCalculatorToVB6()
{
    var calculator = Plugins.FirstOrDefault(p =>
        p.IsLoaded && p.ProgId == "ComPluginDemo.Calculator");
    var vb6Plugin = Plugins.FirstOrDefault(p =>
        p.IsLoaded && p.ProgId == "VB6ComPlugin.StringProcessor");

    if (calculator == null || vb6Plugin == null)
        return;

    try
    {
        dynamic vb6Obj = vb6Plugin.ComObject!;
        vb6Obj.SetCalculatorObject(calculator.ComObject);  // late binding 调用 VB6 方法
        AppendLog($"  [OK] 已将 Calculator COM 对象传递给 {vb6Plugin.Name}");
    }
    catch (Exception ex)
    {
        AppendLog($"  [WARN] 传递 Calculator 对象失败: {ex.Message}");
    }
}
```

**技术要点**:
- C# 使用 `dynamic` 关键字调用 VB6 COM 对象的方法（late binding via IDispatch）
- `SetCalculatorObject` 不是 IComPlugin 接口定义的方法，是 VB6 StringProcessor 独有的
- 传递的是 COM 对象引用（`calculator.ComObject`），不是属性值拷贝

#### 2.3 VB6 端：接收和缓存 COM 对象

```vb
' src/VB6ComPlugin/StringProcessor.cls

' 成员变量
Private m_CalcName As String
Private m_CalcVersion As String
Private m_CalcDescription As String
Private m_CalcObj As Object         ' 保存 C# COM 对象引用

' 由 C# 宿主调用，传入 Calculator COM 对象
Public Sub SetCalculatorObject(ByVal objCalc As Object)
    On Error Resume Next
    Set m_CalcObj = objCalc
    If Not objCalc Is Nothing Then
        m_CalcName = objCalc.Name               ' late binding 读取属性
        m_CalcVersion = objCalc.Version
        m_CalcDescription = objCalc.Description
    End If
    On Error GoTo 0
End Sub
```

**技术要点**:
- VB6 使用 `As Object` 接收 → late binding（通过 IDispatch::Invoke 按 DispId 调用）
- `On Error Resume Next` 防止跨进程 COM 调用异常导致 VB6 崩溃
- 属性值缓存到成员变量，避免后续每次读取都跨 COM 调用

#### 2.4 VB6 端：show 命令弹出窗体

```vb
' StringProcessor.cls Execute 函数中

Case "show"
    If m_CalcObj Is Nothing Then
        Execute = "Error: Calculator object not set. Host must call SetCalculatorObject first."
    Else
        Dim frm As frmCalculator
        Set frm = New frmCalculator
        frm.SetInfo m_CalcName, m_CalcVersion, m_CalcDescription, m_CalcObj
        frm.Show vbModal        ' 模态显示，阻塞到窗体关闭
        Set frm = Nothing       ' 释放窗体对象
        Execute = "Calculator form closed."
    End If
```

#### 2.5 frmCalculator.frm 窗体代码

```vb
' src/VB6ComPlugin/frmCalculator.frm

Private m_CalcObj As Object

Public Sub SetInfo(ByVal sName As String, ByVal sVersion As String, _
                   ByVal sDesc As String, ByVal objCalc As Object)
    txtName.Text = sName
    txtVersion.Text = sVersion
    txtDesc.Text = sDesc
    Set m_CalcObj = objCalc
    If Not m_CalcObj Is Nothing Then
        lblStatus.Caption = "Connected: " & sName & " v" & sVersion
    End If
End Sub

Private Sub btnCalc_Click()
    If m_CalcObj Is Nothing Then
        txtResult.Text = "Error: Calculator not connected"
        Exit Sub
    End If
    If Len(Trim$(txtExpression.Text)) = 0 Then
        txtResult.Text = "Please enter an expression"
        Exit Sub
    End If
    On Error Resume Next
    Dim sResult As String
    sResult = m_CalcObj.Execute(txtExpression.Text)  ' 调用 C# COM 的 Execute
    If Err.Number <> 0 Then
        txtResult.Text = "Error: " & Err.Description
        Err.Clear
    Else
        txtResult.Text = sResult
    End If
    On Error GoTo 0
End Sub
```

### 三、功能二：COM 加载器改进（Manifest 优先 → 注册表回退）

#### 3.1 改进前后对比

**改进前** — 两条策略互斥，manifest 失败直接报错：

```csharp
// 旧逻辑
if (!string.IsNullOrEmpty(entry.ManifestPath))
{
    // 策略 1: 只走 manifest
    if (File.Exists(manifestFullPath))
        return LoadPluginWithActivationContext(info, manifestFullPath);  // 失败就结束
    info.ErrorMessage = $"清单文件不存在: {manifestFullPath}";
    return info;  // 直接返回错误，不尝试注册表
}
// 策略 2: 只走注册表
var type = Type.GetTypeFromProgID(entry.ProgId);
```

**改进后** — manifest 优先，失败回退到注册表：

```csharp
// src/ComPluginHost/Services/ComPluginLoader.cs - 新逻辑

string manifestError = "";

// 策略 1: 优先通过 manifest 免注册加载
if (!string.IsNullOrEmpty(entry.ManifestPath))
{
    var manifestFullPath = Path.IsPathRooted(entry.ManifestPath)
        ? entry.ManifestPath
        : Path.GetFullPath(Path.Combine(baseDir, entry.ManifestPath));

    if (File.Exists(manifestFullPath))
    {
        var result = LoadPluginWithActivationContext(info, manifestFullPath);
        if (result.IsLoaded)
            return result;                               // manifest 成功
        manifestError = result.ErrorMessage ?? "";       // manifest 失败，记录原因
    }
    else
    {
        manifestError = $"清单文件不存在: {manifestFullPath}";
    }
    // ↓↓↓ 不 return，继续尝试注册表 ↓↓↓
}

// 策略 2: 回退到注册表 (也支持 VB6 IDE 调试模式)
var type = Type.GetTypeFromProgID(entry.ProgId);
if (type != null)
{
    var result = CreatePluginFromType(info, type);
    if (result.IsLoaded)
    {
        result.LoadSource = string.IsNullOrEmpty(manifestError)
            ? "注册表"
            : "注册表 (manifest失败回退)";
        return result;
    }
}

// 两种策略都失败
info.ErrorMessage = "所有加载策略均失败:\n" +
    $"  [Manifest] {manifestError}\n" +
    $"  [注册表] 找不到 ProgID: {entry.ProgId}\n" +
    "请确认: ...";
```

#### 3.3 Activation Context API 详解

C# 宿主通过 Win32 API 动态创建激活上下文来实现免注册 COM:

```csharp
// P/Invoke 声明
[DllImport("kernel32.dll", SetLastError = true, CharSet = CharSet.Unicode)]
private static extern IntPtr CreateActCtx(ref ACTCTX pActCtx);

[DllImport("kernel32.dll", SetLastError = true)]
private static extern bool ActivateActCtx(IntPtr hActCtx, out IntPtr lpCookie);

[DllImport("kernel32.dll", SetLastError = true)]
private static extern bool DeactivateActCtx(uint dwFlags, IntPtr lpCookie);

[DllImport("kernel32.dll", SetLastError = true)]
private static extern void ReleaseActCtx(IntPtr hActCtx);

[StructLayout(LayoutKind.Sequential, CharSet = CharSet.Unicode)]
private struct ACTCTX
{
    public int cbSize;
    public uint dwFlags;                   // ACTCTX_FLAG_ASSEMBLY_DIRECTORY_VALID = 0x004
    public string lpSource;                // manifest 文件路径
    public ushort wProcessorArchitecture;
    public ushort wLangId;
    public string? lpAssemblyDirectory;    // 组件搜索目录
    public string? lpResourceName;
    public string? lpApplicationName;
    public IntPtr hModule;
}
```

**加载流程**:
```
1. CreateActCtx(manifest路径, 组件目录)
   → Windows 解析 manifest XML
   → 建立 ProgID → CLSID → DLL 路径的映射
   → 返回激活上下文句柄 hActCtx

2. ActivateActCtx(hActCtx, out cookie)
   → 将上下文压入线程的激活上下文栈
   → 后续 COM 创建调用会优先查找此上下文

3. Type.GetTypeFromProgID("VB6ComPlugin.StringProcessor")
   → 在激活上下文中查找 ProgID 对应的 CLSID
   → 从 manifest 中声明的 DLL 加载 COM 类
   → 不经过注册表

4. Activator.CreateInstance(type)
   → CoCreateInstance 在激活上下文中创建 COM 对象

5. 保持激活上下文活跃 (不 Deactivate)
   → COM 对象生命期内，激活上下文必须存在
   → 句柄和 Cookie 保存在 PluginInfo.ActCtxHandle/ActCtxCookie

6. UnloadPlugin 时释放:
   → DeactivateActCtx(0, cookie)
   → ReleaseActCtx(hActCtx)
   → Marshal.ReleaseComObject(comObject)
```

### 四、功能三：生成 C# COM 类型库 (TLB)

#### 4.1 目的

VB6 中使用**早期绑定** (`Dim obj As CSharpComPlugin.IComPlugin`) 需要类型库。TLB 提供:
- VB6 IDE 中的 IntelliSense 代码补全
- 编译时类型检查
- vtable 直接调用（比 IDispatch 快）

#### 4.2 IDL 接口定义

```c
// src/CSharpComPlugin/CSharpComPlugin.idl
// GUID 必须与 C# 代码中的 [Guid] 属性完全一致

import "oaidl.idl";
import "ocidl.idl";

[
    uuid(A1B2C3D4-E5F6-7890-ABCD-EF1234567890),  // TypeLib GUID
    version(1.0),
    helpstring("CSharpComPlugin 1.0 Type Library")
]
library CSharpComPlugin
{
    importlib("stdole2.tlb");

    [
        object,
        uuid(8C5E2B1A-3F4D-4A6E-9B7C-1D2E3F4A5B6C),  // IComPlugin GUID
        dual,              // 同时支持 vtable (早期) 和 IDispatch (晚期)
        oleautomation,
        helpstring("COM Plugin Interface")
    ]
    interface IComPlugin : IDispatch
    {
        [id(1), propget] HRESULT Name([out, retval] BSTR* pVal);
        [id(2), propget] HRESULT Version([out, retval] BSTR* pVal);
        [id(3), propget] HRESULT Description([out, retval] BSTR* pVal);
        [id(4)]          HRESULT Initialize();
        [id(5)]          HRESULT Execute([in] BSTR input, [out, retval] BSTR* pVal);
        [id(6)]          HRESULT Shutdown();
    };

    [
        uuid(D7E8F9A0-1B2C-3D4E-5F6A-7B8C9D0E1F2A),  // CalculatorPlugin CLSID
        helpstring("Calculator Plugin")
    ]
    coclass CalculatorPlugin
    {
        [default] interface IComPlugin;
    };
};
```

#### 4.3 TLB 生成过程（三次失败，一次成功）

**尝试 1: dscom — 失败（架构不兼容）**
```
$ dscom tlbexport CSharpComPlugin.dll --out CSharpComPlugin.tlb
ERROR: The assembly architecture is not compatible with the current process architecture.
```
原因: dscom.exe 是 x64 (PE32+)，CSharpComPlugin.dll 编译目标是 x86 (`<PlatformTarget>x86</PlatformTarget>`)

**尝试 2: dscom + AnyCPU 构建 — 失败（.NET 8 DLL 始终 PE32）**
```
$ dotnet build -p:PlatformTarget=x64 --output tlbtemp
$ file CSharpComPlugin.dll
PE32 executable... Intel i386 Mono/.Net assembly   ← 仍然是 PE32
```
原因: .NET Core/5+ 的 managed DLL 总是 PE32 格式，平台目标嵌入元数据中

**尝试 3: MIDL + /no_cpp — 失败（缺少 C 预处理器）**
```
$ midl /no_cpp /tlb CSharpComPlugin.tlb CSharpComPlugin.idl
oaidl.idl(29): error MIDL2025: syntax error near "ifndef"
```
原因: `/no_cpp` 跳过预处理器，但 `oaidl.idl` 使用 `#ifndef` 宏

**尝试 4: MIDL + VsDevCmd.bat — 成功**

通过 Python 脚本生成临时 .bat，调用 VsDevCmd.bat 设置 VS 环境后运行 MIDL:

```python
'''
subprocess.run(["cmd.exe", "/c", bat_path], ...)
```

#### 4.4 TLB 使用方式（不需要注册）

VB6 IDE 中: `Project → References → Browse... → 选择 CSharpComPlugin.tlb`

VBP 文件中的引用行格式:
```
Reference=*\G{A1B2C3D4-E5F6-7890-ABCD-EF1234567890}#1.0#0#CSharpComPlugin.tlb#CSharpComPlugin 1.0 Type Library
```

VB6 IDE 查找 TLB 的顺序:

### 五、功能四：VB6 通过 Manifest 免注册加载 C# COM

#### 5.1 问题：VB6 如何在不注册的情况下创建 C# COM 对象

VB6 代码 `CreateObject("ComPluginDemo.Calculator")` 默认查注册表。要免注册，需要让 SxS 激活上下文包含 CSharpComPlugin 的组件清单。

#### 5.2 解决方案：VB6 Manifest 声明依赖

```xml
<!-- src/VB6ComPlugin/VB6ComPlugin.manifest -->
<?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<assembly xmlns="urn:schemas-microsoft-com:asm.v1" manifestVersion="1.0">
  <assemblyIdentity type="win32" name="VB6ComPlugin" version="1.0.0.0" />

  <!-- VB6 COM 类声明 -->
  <file name="VB6ComPlugin.dll">
    <comClass
      clsid="{7A5544D3-6438-4BDB-BBEC-A3C01494BFA5}"
      threadingModel="Apartment"
      progid="VB6ComPlugin.StringProcessor"
      description="StringProcessor Plugin - VB6" />
  </file>

  <!-- 依赖 CSharpComPlugin → 使 VB6 能通过 manifest 创建 C# COM 对象 -->
  <dependency>
    <dependentAssembly>
      <assemblyIdentity type="win32" name="CSharpComPlugin" version="1.0.0.0" />
    </dependentAssembly>
  </dependency>
</assembly>
```

#### 5.3 SxS 依赖解析的目录搜索规则

当 `CreateActCtx` 的 `lpAssemblyDirectory = "VB6ComPlugin目录"` 时，SxS 搜索依赖组件清单的路径:

```
lpAssemblyDirectory/
├── VB6ComPlugin.manifest         ← 主清单
├── VB6ComPlugin.dll
└── CSharpComPlugin/              ← 同名子目录 (SxS 约定)
    ├── CSharpComPlugin.manifest  ← 依赖组件清单 (SxS 在此找到)
    ├── CSharpComPlugin.comhost.dll
    ├── CSharpComPlugin.dll
    ├── CSharpComPlugin.deps.json
    └── CSharpComPlugin.runtimeconfig.json
```

**关键**: SxS 按 `<assemblyIdentity name="CSharpComPlugin">` 在 `lpAssemblyDirectory` 下的同名子目录搜索 `.manifest` 文件。

#### 5.5 frmComExplorer 窗体：免注册创建 C# COM 演示

```vb
' src/VB6ComPlugin/frmComExplorer.frm

Private m_Plugin As Object  ' 晚期绑定，不需要 TLB

' 方式 1: 从 C# 宿主接收已创建的 COM 对象
```

### 八、遇到的问题及详细解决过程

#### 8.1 Python 写 VB6 文件时字符串转义失败

**现象**: 在 Bash 中内联 Python 代码写 StringProcessor.cls，复杂的 VB6 代码（含单引号注释、双引号字符串、`&` 连接符等）导致 Python `SyntaxError: unterminated string literal`

**根因**: Bash → Python → VB6 三层嵌套，引号转义极其复杂

**解决**: 改为独立 Python 脚本文件:
```python
# scripts/write_cls.py
content = """\
VERSION 1.0 CLASS
...完整 VB6 代码...
"""
with open(CLS_PATH, "w", encoding="gbk", newline="\r\n") as f:
    f.write(content)
```

#### 8.2 frmCalculator 中 `Len(Trim)` 语法错误

**现象**: VB6 编译时 `Len(Trim)` 报语法错误

**根因**: `Trim` 在 VB6 中需要参数，`Trim` 不带参数会被解释为函数指针

**修复**: `Len(Trim)` → `Len(Trim$(txtExpression.Text))`

#### 8.3 dscom 导出 TLB 失败

**现象**: `dscom tlbexport CSharpComPlugin.dll` 报 "assembly architecture is not compatible"

**分析过程**:
```
1. dscom.exe = PE32+ (x86-64)
2. CSharpComPlugin.dll = PE32 + x86 target metadata
3. dscom 用 AssemblyLoadContext 加载 → 架构不匹配
4. 尝试 -p:PlatformTarget=AnyCPU 重新编译 → .NET DLL 仍为 PE32（.NET Core 特性）
5. 尝试拷贝到纯 ASCII 路径 → 排除中文路径问题 → 仍失败（确认是架构问题）
```

**解决**: 放弃 dscom，改用 MIDL 从 IDL 文件编译 TLB

#### 8.4 MIDL 在 Bash 环境下运行失败

**现象 1**: 从 Git Bash 直接调用 MIDL，路径参数被 bash 重写

```
midl : command line warning MIDL1009 : unknown argument ignored "I:/"
Processing C:\Program Files\Git\tlb   ← bash 把 /tlb 解释为 Git 安装目录
```

**现象 2**: 加 `/no_cpp` 跳过预处理器 → `oaidl.idl` 中的 `#ifndef` 无法解析

```
oaidl.idl(29) : error MIDL2025 : syntax error near "ifndef"
```

**解决**: 用 Python 生成临时 `.bat` 文件，通过 `VsDevCmd.bat` 设置 VS 环境（提供 `cl.exe` 作为 C 预处理器），再用 `cmd.exe /c` 执行:

```python
bat_content = f'''@echo off
call "{vs_path}\\Common7\\Tools\\VsDevCmd.bat" -arch=x86 >nul 2>&1
midl /tlb "{TLB_PATH}" /I "{sdk_inc}\\um" /I "{sdk_inc}\\shared" "{idl_path}"
'''
subprocess.run(["cmd.exe", "/c", bat_path], ...)
```

#### 8.5 VB6 IDE "Missing reference: CSharpComPlugin 1.0 Type Library"

**现象**: 打开 VBP 后 VB6 IDE 报找不到类型库引用

**分析**:
```
VBP 引用行: Reference=*\G{A1B2C3D4-...}#1.0#0#CSharpComPlugin.tlb#...

VB6 IDE 查找顺序:
1. 按 GUID 查注册表 HKCR\TypeLib\{A1B2C3D4-...} → 未注册 → 失败
2. 按文件路径 CSharpComPlugin.tlb (相对于 VBP 目录) → 文件存在但步骤1已报错
```

**解决方案**:
- 方案 A: `Project → References → Browse → CSharpComPlugin.tlb` 手动添加（推荐）
- 方案 B: 去掉 TLB 引用，改用晚期绑定 `As Object`（当前采用）
- 方案 C: 注册 TLB（`pythoncom.RegisterTypeLib()`），但违反免注册原则

#### 8.6 VBP 中 Form 行丢失

**现象**: 之前通过脚本添加的 `Form=frmCalculator.frm` 行消失了

**根因**: 用户在 VB6 IDE 中打开并保存了项目，VB6 IDE 如果找不到 .frm 文件（或文件有问题），会在保存时去掉该 Form 行

**解决**: Python 脚本检测并重新添加:
```python
if not any("frmCalculator" in l for l in lines):
    # 在 Class=StringProcessor 行后面插入
    new_lines.append("Form=frmCalculator.frm\n")
```

### 十一、VB6 热调试环境完善（Activation Context API）

#### 11.1 问题背景

之前的调试方案（第 3.4 节）存在一个限制：VB6 IDE 调试模式下，frmComExplorer 的 **"Create via Manifest"** 按钮无法工作。

原因分析：
```
编译后 DLL 模式:
  C# Host 进程 → 加载 VB6ComPlugin.manifest → <dependency> CSharpComPlugin
  → SxS 激活上下文已包含 CSharpComPlugin 的 comClass 声明
  → VB6 代码中 CreateObject("ComPluginDemo.Calculator") → 成功 ✓

VB6 IDE 调试模式:
  VB6 IDE 进程 (VB6.EXE) → 没有 SxS 激活上下文
  → VB6 代码中 CreateObject("ComPluginDemo.Calculator")
  → 查注册表 → CSharpComPlugin 未注册 → 失败 ✗
```

根本原因：VB6 IDE 按 F5 调试 ActiveX DLL 时，VB6 代码运行在 **VB6 IDE 进程**中（通过 DCOM/代理与客户端通信），而 C# 宿主的 SxS 激活上下文在**另一个进程**中，无法跨进程传递。

#### 11.2 解决方案：VB6 端手动激活 Manifest

新增 `modActCtx.bas` 模块，封装 Win32 Activation Context API，让 VB6 代码能自行激活 CSharpComPlugin.manifest：

```
frmComExplorer.btnCreateViaManifest_Click:
  │
  ├─ Step 1: 直接 CreateObject (编译 DLL 模式，SxS 已激活)
  │   → 成功? → 使用对象
  │   → 失败? → 继续 Step 2
  │
  └─ Step 2: 手动 Manifest 激活 (VB6 IDE 调试模式)
      ├─ FindCSharpManifest() → 搜索多个路径找 manifest
      ├─ ActivateManifestCtx(manifestPath) → CreateActCtxW + ActivateActCtx
      ├─ CreateObject("ComPluginDemo.Calculator") → 成功!
      └─ 保持上下文活跃直到窗体关闭 → Form_QueryUnload 中 DeactivateManifestCtx
```

##### modActCtx.bas — Win32 Activation Context API 模块

```vb
' src/VB6ComPlugin/modActCtx.bas

' Win32 API 类型声明 (32-bit, 结构体 32 字节)
Private Type ACTCTX
    cbSize As Long                    ' 结构体大小 = 32
    dwFlags As Long                   ' 标志位
    lpSource As Long                  ' 清单文件路径指针 (LPCWSTR)
    wProcessorArchitecture As Integer ' 处理器架构
    wLangId As Integer                ' 语言 ID
    lpAssemblyDirectory As Long       ' 组件搜索目录指针 (LPCWSTR)
    lpResourceName As Long            ' 资源名称指针
    lpApplicationName As Long         ' 应用名称指针
    hModule As Long                   ' 模块句柄
End Type

' Win32 API 声明
Private Declare Function CreateActCtxW Lib "kernel32" (pActCtx As ACTCTX) As Long
Private Declare Function ActivateActCtx Lib "kernel32" ( _
    ByVal hActCtx As Long, lpCookie As Long) As Long
Private Declare Function DeactivateActCtx Lib "kernel32" ( _
    ByVal dwFlags As Long, ByVal ulCookie As Long) As Long
Private Declare Sub ReleaseActCtx Lib "kernel32" (ByVal hActCtx As Long)

' 模块级状态
Private m_hActCtx As Long     ' 激活上下文句柄
Private m_Cookie As Long      ' 激活 Cookie (用于 Deactivate)
Private m_Active As Boolean   ' 是否已激活

' 公共变量: 宿主传递的基目录
Public g_ManifestBaseDir As String
```

关键技术点：
- 使用 `CreateActCtxW` (Unicode W 版本) 而非 `CreateActCtxA`
- 结构体中字符串字段声明为 `Long` (指针)，通过 `StrPtr()` 传递 VB6 BSTR 的 Unicode 指针
- VB6 BSTR 内部是 Unicode，与 W 版 API 直接兼容
- `ACTCTX_FLAG_ASSEMBLY_DIRECTORY_VALID = &H4` 告诉 SxS 使用指定目录搜索依赖组件

##### FindCSharpManifest() — 多路径搜索

```vb
Public Function FindCSharpManifest() As String
    Dim sPaths(3) As String

    ' Path 0: 宿主传递的 BaseDir (最可靠，VB6 IDE 调试模式)
    '   C# 宿主调用 SetManifestBaseDir(BaseDir) 设置
    If Len(g_ManifestBaseDir) > 0 Then
        sPaths(0) = g_ManifestBaseDir & "CSharpComPlugin\CSharpComPlugin.manifest"
    End If

    ' Path 1: App.Path\CSharpComPlugin\ (编译 DLL 被宿主加载时)
    sPaths(1) = App.Path & "\CSharpComPlugin\CSharpComPlugin.manifest"

    ' Path 2: App.Path\VB6ComPlugin\CSharpComPlugin\ (宿主带 VB6 子目录)
    sPaths(2) = App.Path & "\VB6ComPlugin\CSharpComPlugin\CSharpComPlugin.manifest"

    ' Path 3: CurDir$\CSharpComPlugin\ (VB6 IDE 的当前目录可能是项目目录)
    sPaths(3) = CurDir$ & "\CSharpComPlugin\CSharpComPlugin.manifest"

    ' 按优先级搜索
    Dim i As Long
    For i = 0 To 3
        If Len(sPaths(i)) > 0 Then
            If Len(Dir$(sPaths(i))) > 0 Then
                FindCSharpManifest = sPaths(i)
                Exit Function
            End If
        End If
    Next i

    FindCSharpManifest = ""  ' 未找到
End Function
```

搜索路径优先级说明：
| 优先级 | 路径来源 | 适用场景 |
|--------|---------|---------|
| 0 | `g_ManifestBaseDir` (宿主传入) | VB6 IDE 调试模式（最可靠） |
| 1 | `App.Path\CSharpComPlugin\` | 编译 DLL 被宿主直接加载 |
| 2 | `App.Path\VB6ComPlugin\CSharpComPlugin\` | 编译 DLL 在 VB6 子目录中 |

##### ActivateManifestCtx() — 激活清单

```vb
Public Function ActivateManifestCtx(ByVal sManifestPath As String) As Boolean
    If m_Active Then
        ActivateManifestCtx = True   ' 已激活，直接返回
        Exit Function
    End If

    Dim ctx As ACTCTX
    ctx.cbSize = 32                  ' sizeof(ACTCTX) = 32 bytes (32-bit)
    ctx.lpSource = StrPtr(sManifestPath)  ' Unicode 字符串指针

    ' 设置组件搜索目录为 manifest 所在目录
    Dim sDir As String
    sDir = Left$(sManifestPath, InStrRev(sManifestPath, "\") - 1)
    ctx.dwFlags = ACTCTX_FLAG_ASSEMBLY_DIRECTORY_VALID  ' &H4
    ctx.lpAssemblyDirectory = StrPtr(sDir)

    ' 创建激活上下文
    m_hActCtx = CreateActCtxW(ctx)
    If m_hActCtx = INVALID_HANDLE_VALUE Then  ' -1
        ActivateManifestCtx = False
        Exit Function
    End If

    ' 激活: 压入线程的激活上下文栈
    If ActivateActCtx(m_hActCtx, m_Cookie) = 0 Then
        ReleaseActCtx m_hActCtx
        m_hActCtx = 0
        ActivateManifestCtx = False
        Exit Function
    End If

    m_Active = True
    ActivateManifestCtx = True
End Function
```

##### frmComExplorer.frm — btnCreateViaManifest_Click 两步策略

```vb
Private Sub btnCreateViaManifest_Click()
    On Error Resume Next

    ' Step 1: 直接 CreateObject (编译 DLL 模式，SxS 激活上下文已存在)
    Dim obj As Object
    Set obj = CreateObject("ComPluginDemo.Calculator")

    If Err.Number = 0 Then
        GoTo CreateSuccess       ' 成功!
    End If

    Err.Clear
    On Error GoTo 0

    ' Step 2: 手动 Manifest 激活 (VB6 IDE 调试模式)
    Dim sManifest As String
    sManifest = FindCSharpManifest()       ' modActCtx.bas

    If Len(sManifest) = 0 Then
        lblStatus.Caption = "Status: Cannot find CSharpComPlugin.manifest"
        Exit Sub
    End If

    If Not ActivateManifestCtx(sManifest) Then   ' modActCtx.bas
        lblStatus.Caption = "Status: Failed to activate manifest"
        Exit Sub
    End If

    On Error Resume Next
    Set obj = CreateObject("ComPluginDemo.Calculator")   ' 重试
    If Err.Number <> 0 Then
        DeactivateManifestCtx
        Exit Sub
    End If
    On Error GoTo 0

CreateSuccess:
    Set m_Plugin = obj
    obj.Initialize
    lblStatus.Caption = "Status: Created via Manifest (Reg-Free COM)"
    RefreshProperties
End Sub

Private Sub Form_QueryUnload(Cancel As Integer, UnloadMode As Integer)
    Set m_Plugin = Nothing
    ' 清理手动激活的上下文
    If IsManifestActive Then DeactivateManifestCtx
End Sub
```

#### 11.8 工作原理总结

```
┌─────────────────────────────────────────────────────────────────┐
│         VB6 IDE 调试模式下 "Create via Manifest" 原理           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  VB6 IDE 进程 (VB6.EXE)                                        │
│  ├─ 没有 SxS 激活上下文 (不像 C# 宿主有 app.manifest)           │
│  ├─ CreateObject("ComPluginDemo.Calculator")                    │
│  │   → 查注册表 → CSharpComPlugin 未注册 → 失败                 │
│  │                                                              │
│  ├─ modActCtx.FindCSharpManifest()                              │
│  │   → g_ManifestBaseDir = "C:\...\bin\Debug\net8.0-windows\"   │
│  │   → 检查 g_ManifestBaseDir\CSharpComPlugin\CSharpComPlugin.manifest │
│  │   → 文件存在 ✓                                               │
│  │                                                              │
│  ├─ modActCtx.ActivateManifestCtx(manifestPath)                 │
│  │   ├─ CreateActCtxW(ACTCTX{                                   │
│  │   │     lpSource = "...\CSharpComPlugin\CSharpComPlugin.manifest",│
│  │   │     lpAssemblyDirectory = "...\CSharpComPlugin\"          │
│  │   │   })                                                     │
│  │   │   → Windows 解析 manifest → 建立 ProgID→CLSID→DLL 映射   │
│  │   │   → 返回 hActCtx                                         │
│  │   └─ ActivateActCtx(hActCtx, &cookie)                        │
│  │       → 激活上下文压入线程栈                                   │
│  │                                                              │
│  ├─ CreateObject("ComPluginDemo.Calculator")  ← 重试            │
│  │   → 在激活上下文中查找 → 找到 CLSID                           │
│  │   → 加载 CSharpComPlugin.comhost.dll → 创建 COM 对象         │
│  │   → 成功! ✓                                                   │
│  │                                                              │
│  └─ Form_QueryUnload:                                           │
```
